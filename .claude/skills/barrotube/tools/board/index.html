설계문서

1. 개요 & 범위

BarroTube는 경제 채널 econ-daily의 에피소드를 브리프 → 대본 → 자산 → 렌더 → QA → 메타 → 승인 → 발행까지 자동 산출하는 파이프라인이다. 이 문서는 그 파이프라인의 단일 진실 원천이자, 위 보드를 통해 실제로 실행하는 콘솔이다.

  • In scope — 에피소드 생성·상태 조회·단계별 실행(대본/자산/렌더/QA/메타/승인/발행)
  • Out of scope — 시리즈 기획(CEO 에이전트), 마케팅 분석, 재생목록 관리(S12는 발행 후처리 훅)
브리지가 필요합니다. 이 문서는 node tools/board/server.js가 띄운 로컬 서버(기본 http://127.0.0.1:8933)에서 열어야 목록·실행이 동작합니다. 파일을 직접 더블클릭하면 문서 부분만 보입니다.

4. 상태 모델

.episode_status.json에 current_stage / status / stage_history[]가 기록된다.

관측된 status의미
created브리프만 생성됨
in_progress / completed해당 스테이지 진행/완료
failed스테이지 실패 (stage_history[].error 확인)
awaiting_approval / in_reviewS10 보드 승인 대기
published / scheduled발행 완료 / 예약
deferred / blocked보류 / 차단
playlist_registered / playlist_pendingS12 재생목록 처리
status 파일을 100% 믿지 마세요. 발행 후 갱신이 누락된 EP가 있습니다(예: EP-0059는 S9b/completed인데 실제로는 발행됨). 이 보드는 80_publish_result.json 존재 여부로 발행을 판정합니다. 또 .episode_status.json 자체가 없는 EP도 있습니다.

5. QA 체크리스트 (발행 전)

  1. 사양: shorts = 1080×1920 · 9:16 · H.264 · 58~62초 (ffprobe로 확인)
  2. 오디오: 내레이션 라우드니스 ≈ −14 LUFS · BGM이 나레이션을 덮지 않음(사이드체인 덕킹)
  3. 자막: 전 씬 노출 · 오탈자 없음 · 하단 UI 세이프존 침범 없음
  4. 이미지 텍스트: AI 생성 한글은 오탈자 상습 → 인트로/썸네일 타이포는 확대 검수 (실사례: "메타"→"머타")
  5. 씬-내레이션 일치: 렌더된 클립이 해당 씬 내용인지 (다운로드 순서 꼬임으로 엉뚱한 영상이 섞인 사고 있었음)
  6. 팩트: 수치·인용은 35_factcheck.md와 대조 · 단서 조건(예: "외국 기업 기준") 삭제 금지
  7. 고지: 설명란에 "투자 조언 아님" 문구 포함

6. CLI 레퍼런스

모두 node scripts/automation/<script>. 보드의 단계 버튼이 그대로 이 명령들을 실행한다.

단계명령
S0 생성create-episode.js --channel econ-daily --topic "주제" --format shorts --length 60
S4 대본generate-script.js --episode <dir> --platform shorts
S5 팩트체크run-factcheck.js --episode EP-YYYY-NNNN --platform shorts
S6a TTSgenerate-tts.js --script <30_script.md> --out-dir <40_assets/tts>
S6b 길이 동기화sync-durations.js --script <30_script.md> --tts-dir <40_assets/tts>
S6c 씬 이미지generate-image-gemini.js --script <30_script.md> --out-dir <40_assets/images> --aspect 9:16
S6d 인트로generate-intro.js --episode <dir>
S6e 썸네일generate-thumbnail.js --episode <dir> --platform shorts
엔드카드generate-endcard.js --episode <epdir> --platform shorts --base <마지막 씬.png>
S7 렌더render-direct.js --episode <dir> --out <55_render/video.mp4>
S8 QAgenerate-qa-report.js --episode <dir> --platform shorts
S9 메타generate-metadata.js --episode <dir> --platform shorts
S10 승인approve-episode.js --episode EP-YYYY-NNNN --platform shorts --by <name>
S11 발행publish-youtube.js --video <mp4> --meta <70_publish_meta.json> --thumbnail <47_thumbnail.jpg>
썸네일은 JPG로. YouTube 썸네일 한도는 2MB인데 생성 PNG는 흔히 2MB를 넘습니다(invalidImage 에러). sips -s format jpeg로 변환한 47_thumbnail.jpg를 넘기세요.

8. 리스크 & 함정 (실사고 기록)

  • TTS 길이 ±15~20% 변동 → 대본 target_seconds를 믿지 말고 sync-durations.js로 실측 반영.
  • 미디어 다운로드 레이스 — 브라우저 다운로드가 끝나기 전에 "Downloads 최신 파일"을 집으면 이전 영상이 섞여 들어간다. 파일 도착을 폴링으로 확인하고, 렌더 후 컨택트시트로 씬 순서를 눈으로 검증할 것.
  • AI 한글 타이포 오탈자 — 인트로/썸네일 글자는 확대 검수 후 통과.
  • 발행은 되돌리기 어렵다 — 이 보드의 발행 버튼은 확인 토큰(PUBLISH)을 요구한다. 유튜브는 업로드된 영상의 파일 교체가 불가하므로, 영상 수정 = 내리고 재업로드(조회수 초기화).
  • OAuth 리프레시 토큰 만료 — invalid_grant가 뜨면 setup-youtube-oauth.js로 재인증.
  • produce-episode.js는 --execute 없이도 자산을 생성(=비용)할 수 있다. 보드는 이 스크립트를 화이트리스트에 넣지 않았다.
